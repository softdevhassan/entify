Developer Manual

A comprehensive guide for setting up the environment, installing dependencies, and navigating the codebase.

Step 1: The Environment

Editor Installation

We recommend using Visual Studio Code (VS Code). It’s the industry standard and makes working with Python much easier. Once installed, make sure to add the "Python" extension by Microsoft from the Marketplace to get full support for the code.

Python Installation

Download and install Python 3.12+ from the official website. During installation on Windows, it is very important to check the box that says "Add Python to PATH". This allows you to run python commands from any terminal or command prompt.

Verification Command

python --version

Step 2: Configuration

Once Python is ready, you need to set up a clean space for the project. Using a virtual environment is the best way to keep your project packages separate from your main system.

Setup Commands

# 1. Create environment python -m venv venv
# 2. Activate (Windows) venv\Scripts\activate
# 3. Install Packages pip install -r requirements.txt

Note: Installing the packages can take a few minutes as it downloads spaCy and the CRF libraries. If you encounter any error, make sure your internet is stable and your pip is up to date: python -m pip install --upgrade pip.

Step 3: Understanding the Structure

The codebase is organized logically to separate the AI logic from the web interface. Here is a map of where everything lives.

/models

The brain house. Contains model trainers, joblib files, and the CRF features script.

/templates

All the HTML files. We use Jinja2 to render these pages dynamically.

/static

The styling and script unit. Holds standard CSS, JavaScript, and project assets.

app.py

The core entry point. It manages all the routes and the API bridge for analysis.

requirements.txt

The shopping list of all Python libraries needed to run the platform.

llms.txt

A humanized summary of the project specs for better indexing.

Running for Production

Once the folders are in place and dependencies are installed, you simply launch the engine. The app defaults to 127.0.0.1:5000, allowing you to debug and test your changes instantly.

python app.py
